1) Présentation du projet

WeatherApp Master est un projet de consultation météo orienté usage quotidien : l'utilisateur obtient rapidement la température, la localisation et la qualité de l'air, avec une interface simple.

Côté technique, l'application s'appuie sur Node.js pour exposer les routes et centraliser les appels API. Les données importantes sont enregistrées dans SQLite pour garder une trace des relevés et illustrer un flux complet front/back.

4) Éléments importants

  • Détection de la position utilisateur pour charger automatiquement la météo locale.
  • Récupération des données via API externe (météo + qualité de l'air) et affichage instantané.
  • Enregistrement des relevés dans SQLite via une route API Node.js.
  • Consultation de l'historique pour suivre l'évolution des conditions.
  • Interface responsive pensée pour une lecture rapide des informations utiles.

5) Architecture côté serveur

Le serveur gère les routes principales, appelle les services météo et enregistre les données utiles pour assurer la cohérence entre affichage en direct et historique.

6) Expérience utilisateur

L'interface met en avant les informations météo essentielles et reste simple à lire, sur ordinateur comme sur mobile.

8) Conclusion

Ce projet montre ma capacité à construire une application web complète, de la collecte de données externes jusqu'au stockage et à l'affichage côté utilisateur. Il combine des bases solides en API, JavaScript et architecture Node.js.
